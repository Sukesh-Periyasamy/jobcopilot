ARCHITECTURE
============
- Split architecture: Frontend (GitHub Pages) + Backend (Render Free) + Database (MongoDB Atlas Free)
- Render Cron Job for daily automated scraping at 08:00 IST
- Telegram Bot API for notifications
- Zero build step frontend (pure HTML/CSS/JS)

FEATURES IMPLEMENTED
====================
• Multi-source job scraping (LinkedIn, Indeed, Naukri, Google via JobSpy)
• Automatic deduplication via MongoDB unique index on job_url
• Advanced filtering (source, location, company, keyword, job type, date range)
• Full-text search across titles and descriptions
• Job saving and bookmarking
• Application status tracking (Interested → Applied → Interview → Offer → Joined/Rejected)
• Company watchlist (max 50, default: Siemens Healthineers, GE HealthCare, Philips, Medtronic, Abbott, Dozee)
• Telegram notifications (daily summary + watchlist alerts, message splitting at 4096 chars)
• Dashboard metrics (total jobs, today, this week, saved, applied)
• Pagination (50 per page, configurable)
• Fresh job badges (🟢 Posted Today, 🟡 Posted This Week, ⚪ Older)
• Retry with exponential backoff (all external calls)
• Rotating logs (10MB, 5 backups, ISO 8601 format)
• Configurable via .env (search terms, locations, sources, schedule time)
• Dark mode glassmorphism UI (Vercel/Linear/GitHub inspired)
• Mobile-responsive design
• Loading skeletons and empty states
• Toast notifications

FREE-TIER COMPLIANCE
=====================
✅ Render Free: single web service + single cron job, 512MB RAM
✅ MongoDB Atlas Free: 512MB storage, 7 indexes (well under 64 limit)
✅ GitHub Pages: static frontend, no build step
✅ In-memory exports: no disk I/O (BytesIO buffers)
✅ Sequential scraper execution: fits within 15-min cron timeout
✅ No paid dependencies or services added
